## Особливості files_fs-імплементації

Функція New() для files_fs-імплементації приймає root-каталог явно. Всі операції мають бути обмежені цим root-каталогом.

Для files_fs:

* Абсолютні path-и приймаються тільки якщо вони знаходяться всередині root; у Data.Path повертається абсолютний path.
* Clean(root) очищає вміст root, але не видаляє сам root-каталог.
* CRC не рахуємо і залишаємо nil.
* CreatedAt залишаємо zero, бо стандартний Go API не дає портативного часу створення файлу.
* MimeType для файлів визначаємо через стандартну Go-бібліотеку mime за розширенням.
* Symlink-и не розіменовуємо і позначаємо MimeType = "inode/symlink".
* depth=0 у List() означає рекурсивний обхід без обмеження; depth=1 — тільки прямі нащадки; depth<0 — помилка.
* Remove(path, false) видаляє тільки сам файл або порожній каталог; Remove(path, true) може видаляти каталог рекурсивно.
